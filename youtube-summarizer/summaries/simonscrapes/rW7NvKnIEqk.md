# Claude just got another superpower (but they kept it quiet!)

Video ID: `rW7NvKnIEqk`

## Summary
This video covers a set of newly released (but under-announced) features in the Claude desktop app that significantly upgrade its capability for structured, multi-session project work. The creator argues that these updates effectively close the gap between the Claude terminal/VS Code workflow and the desktop app, making it viable to run a full agentic operating system natively in the UI. The video is most relevant to power users, product managers, freelancers, and agency operators who use Claude Code for complex, ongoing business work and have been relying on the terminal for flexibility.

## Key insights
- **The core problem with Claude desktop before this update:** Everything was isolated — single chats, no cross-session visibility, no file access, and no sense of progress across projects. This made it unsuitable for real, multi-faceted business workflows.
- **Multiple sessions view:** Claude desktop now shows all active sessions in a holistic dashboard rather than a simple chat list on the left. Users can see how sessions connect and manage multiple tasks tied to a single project simultaneously.
- **Split view (up to 4 sessions at once):** Users can drag and drop sessions into a split-screen layout, running up to four independent windows simultaneously. Each window can display a chat, a plan, or files — fully customizable use of screen real estate.
- **Sidebar plan viewer:** Claude now renders project plans in formatted, collapsible Markdown in a dedicated sidebar panel. Previously, users had to scroll up through the chat to reference the plan; now it sits side-by-side with the active session for real-time progress tracking.
- **File viewer with Markdown rendering:** Users can now browse the underlying file structure and open output files (especially Markdown) directly in the desktop app with rendered previews. This eliminates the need to open a separate editor like VS Code just to view formatted outputs.
- **File structure visibility:** Previously, the file system behind Claude Code was completely invisible in the desktop app. Now users can see folders, navigate into them, read and edit files like `CLAUDE.md`, and access brand context, agent rules, and client-specific materials.
- **Folder-scoped sessions:** When starting a new Claude Code session, users can now choose which folder to initialize in — meaning they can drop directly into a specific client folder with all the relevant context pre-loaded, rather than starting in a generic isolated environment.
- **Agentic OS compatibility:** The creator's pre-built "agentic operating system" (a structured file system designed to store, load, and improve context across sessions) now works natively in the desktop app — something that previously required the terminal due to file access limitations.
- **Known limitation 1 — PNG/image rendering:** Files like PNGs generated by tools (e.g., Excalidraw skill) display as raw code rather than rendered images. Expected to be fixed.
- **Known limitation 2 — Hidden/skills folders:** Hidden files and skills folders (e.g., custom agent skill definitions) are not yet visible or editable in the desktop app. Users must still go to the terminal or VS Code to modify these.
- **Known limitation 3 — `.env` files:** Environment variable files storing API credentials cannot currently be viewed or edited in the Claude desktop app. However, Claude Code can still *access* and *use* those credentials for API calls — users just can't modify them from within the app yet.
- **Overall trajectory:** The creator expresses genuine surprise at how quickly Claude has reached this stage and says they can now "see themselves fully moving away from VS Code."

## Use cases
- **Agency operators** managing multiple clients, each with their own folder, brand context, and active projects — now able to navigate between client workspaces cleanly in one UI.
- **Freelancers or consultants** running parallel workstreams for different clients or projects who need to monitor progress across sessions without switching tools.
- **Product managers or builders** who maintain structured project plans in Markdown and need to track execution against those plans in real time.
- **Teams building agentic systems** (custom Claude workflows with rules, context files, and output directories) who previously had to use the terminal to interact with the file system.
- **Anyone who was using Claude Code in VS Code terminals** primarily for file visibility and now wants a cleaner, GUI-based alternative.
- **Content or marketing teams** storing brand voice profiles, positioning documents, and client briefs as context files — now able to preview and edit those directly in the desktop app.

## Patterns & frameworks

### Agentic Operating System (Agentic OS)
A pre-built file structure the creator developed to give Claude persistent, structured business context across sessions. It works by:
1. Organizing files into client folders and project subfolders
2. Storing reusable context (brand voice, positioning, agent rules) in defined locations
3. Loading only the relevant context when a session starts in a specific folder
4. Designed to improve its own outputs over time through consistent context management
Previously required terminal access; now compatible with the Claude desktop app natively.

### Folder-Scoped Context Loading
A repeatable pattern where each new Claude Code session is initialized inside a specific client or project folder, so only the relevant context is active. This avoids polluting sessions with unrelated information and mirrors how a human would "open a client file" before starting work.

### Split-View Multi-Session Workflow
A operational pattern for parallel work: run up to four independent Claude sessions simultaneously, each in its own panel. One might show a plan, another an active chat, another a file preview — enabling monitoring and execution of complex projects without context switching between apps.

### Plan-Then-Execute with Sidebar Tracking
A structured workflow pattern where Claude generates a Markdown plan at the start of a session, which is then pinned in the sidebar. The user executes tasks in the chat while referencing the plan in real time — separating planning artifacts from execution conversation to maintain clarity and progress visibility.
