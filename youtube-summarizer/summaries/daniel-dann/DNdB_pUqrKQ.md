## Patterns & frameworks
**Create with AI Workflow (Beautiful.ai 3.0)**
A linear, prompt-to-deck pipeline: (1) write or speak a prompt with optional file uploads for context, (2) review and edit the AI-generated outline before any design is applied, (3) select visual direction (theme, image source, style), (4) generate the full designed deck in real time. The key principle is that intent is preserved at every stage rather than requiring the user to course-correct after generation.

**Smart Slides**
Pre-built, data-driven interactive slide formats (charts, timelines, graphs, team layouts) that auto-adapt their visual styling to match the deck's theme. The pattern: you supply the data, the platform handles the visual rendering and theme consistency automatically.

**Theme-First Brand System**
A reusable configuration layer where typography, colors, shapes, decorations, footers, and default layouts are set once at the deck level and applied globally. Themes are saved to a library for reuse across future presentations — a scalable pattern for teams with brand guidelines.

**Slide AI (Per-Slide Regeneration)**
A fallback loop pattern within the editor: if any individual slide misses the mark, it can be regenerated or replaced in isolation without affecting the rest of the deck. This prevents the common failure mode of being locked into a bad AI output.

**Shareable Link with Analytics**
A distribution and feedback loop pattern: share the deck via a controlled link (with optional password, expiration, or viewer auth), then use built-in slide-level engagement analytics to identify which sections land and which need revision before the next use.
