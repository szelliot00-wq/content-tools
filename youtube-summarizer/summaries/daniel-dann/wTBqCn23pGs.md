## Patterns & frameworks

**Traditional SEO + AI Visibility Dual-Track Framework**
The core mental model of the video: SEO strategy must now operate on two tracks simultaneously — traditional ranking signals (keywords, backlinks, technical health) and AI citation signals (entity clarity, identity markup, structured content). Semrush One is positioned as the tool that addresses both tracks under one roof.

**MCP (Model Context Protocol) Integration Pattern**
A workflow pattern where an AI agent inside a code editor is connected to a live third-party data source (Semrush) via MCP. Instead of the agent relying on general internet knowledge, it queries a proprietary, real-time database. The pattern: authorize connection → load tools → issue natural-language prompts → receive structured, data-backed outputs (JSON reports, HTML, articles).

**Competitor Reverse-Engineering Workflow**
A repeatable process demonstrated with bbc.com: (1) select a dominant competitor domain, (2) run a technical/entity-level audit using live data, (3) extract the structural patterns behind their AI search performance, (4) use those patterns as the design specification for your own content. Essentially: study the leader, extract the formula, replicate the structure.

**Data-First Content Creation**
A shift away from the traditional "write then optimize" workflow. The pattern here is: pull real market data → run gap analysis → feed findings directly into content generation → publish content already aligned with performance benchmarks. The optimization is baked in from the start, not retrofitted.

**Entity Signal Implementation**
A specific SEO pattern highlighted in the video: to appear in AI-generated answers, content must contain explicit identity signals (e.g., `NewsMediaOrganization` schema markup, structured entity tagging). The agent automatically identifies missing signals from the audit and adds them to the codebase — making this a closed-loop, self-correcting workflow.
