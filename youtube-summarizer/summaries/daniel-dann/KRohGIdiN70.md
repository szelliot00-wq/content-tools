## Patterns & frameworks

**1. Role-Specific AI Helper Model**
Rather than one general assistant, Sintra assigns distinct AI helpers to defined business functions (content, web, operations). Each helper is scoped to its role, which keeps outputs structured and relevant. The pattern mirrors how a real team is organized by function rather than having one generalist handle everything.

**2. Define Once, Apply Everywhere (Shared Context Architecture)**
Users input their business profile once at setup (industry, team size, revenue, customer definition, business description). This context persists and informs all helpers across all tasks. The framework eliminates prompt repetition and keeps outputs aligned with the user's actual business rather than generic defaults.

**3. Progressive Refinement Chain (Single Chat, Multiple Outputs)**
Demonstrated throughout the Soshi walkthrough: start with a broad request (10 titles) → refine for audience and SEO → extract thumbnail hooks → write a description → repurpose for social → build a content calendar → identify differentiation angles → create a reusable workflow template. Each step builds on the previous one inside the same conversation, creating a compounding output without context loss.

**4. Awareness-to-Authority Content Structure**
The 30-day content plan generated by Soshi is organized around a directional arc: early content builds audience awareness, later content establishes channel authority. This is a named strategic pattern for content channels that moves beyond random topic generation toward deliberate audience development.

**5. Job-to-Be-Done Differentiation Framework**
One of the five unique channel angles suggested by Soshi is evaluating AI tools based on the "job to be done" rather than brand reputation or feature lists. This is a well-established product thinking framework (popularized by Clayton Christensen) applied here to content differentiation — reviewing tools through the lens of what specific problem they actually solve for the user.

**6. Conversion Section Breakdown Model**
Comet's "conversion breakdown document" assigns a psychological function to each section of a landing page (e.g., headline = loss aversion, proof elements = credibility signaling). This functions as a structured copywriting audit framework that explains *why* each element exists, not just *what* it says — useful for iterating on conversion logic systematically.

**7. Reusable Template Output Pattern**
Both helpers (Soshi and Comet) are prompted to reformat outputs into copy-paste, reusable templates (Notion-style or Webflow-ready). This pattern treats AI-generated content not as a one-time answer but as a repeatable system asset, shifting the value from a single output to an ongoing operational tool.
