# The Promptware Kill Chain: How Prompt Injection Becomes AI Malware

Video ID: `K68sqG18270`

## Summary
The video introduces "promptware" — a new class of malware that exploits generative AI systems through prompt injection rather than traditional code execution. It walks through a kill chain adapted from cybersecurity frameworks, showing how an attacker can move from initial access through reconnaissance, persistence, lateral movement, and ultimately real-world impact. The core argument is that LLMs' architectural inability to separate instructions from data makes them fundamentally vulnerable, and that defenders must adopt a zero-trust posture rather than waiting for vendors to patch the problem.

## Key insights
- **The architectural root cause is instruction/data conflation.** Traditional systems enforce a boundary between code and data; LLMs treat everything as tokens, so a malicious instruction inside an email, document, or image carries the same authority as a system prompt.
- **Prompt injection is the entry point** — either direct (attacker types commands into the AI) or indirect (attacker plants payloads in content the AI later consumes, like product reviews or calendar invites).
- **Privilege escalation maps to jailbreaking.** Techniques like "ignore all previous rules," roleplay scenarios, and persona shifts trick the model into bypassing its safety alignment, effectively granting attacker-level access to the reasoning engine.
- **Recon happens *after* compromise**, not before — the model is manipulated into revealing its own tools, APIs, plugins, and connected systems, exposing its attack surface from the inside.
- **Persistence exploits long-term memory.** By planting malicious prompts in RAG databases, email archives, or chat histories, attackers cause the system to re-execute the payload every time that data is referenced — the infection becomes self-reinforcing.
- **Command-and-control is achieved via the LLM's own internet access**, turning promptware from a static threat into a dynamically updatable one where attackers can push new instructions remotely.
- **Lateral movement is amplified by AI integration.** Because agents connect to email, calendars, enterprise tools, smart devices, and other agents, a single compromised agent can propagate the payload across an entire ecosystem — analogous to a self-replicating worm.
- **The end goal is real-world impact:** data theft, financial fraud, cryptocurrency transfers, or even arbitrary code execution if the agent has system access.
- **Defense requires a zero-trust posture.** Assume breach, treat AI agents as untrusted execution environments, limit tool permissions, detect persistence mechanisms, and design systems to break the kill chain at every link — rather than relying on vendors to fix a fundamentally architectural problem.
