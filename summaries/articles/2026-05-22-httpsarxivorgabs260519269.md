# CODA: Rewriting Transformer Blocks as GEMM-Epilogue Programs

Source: https://arxiv.org/abs/2605.19269

## Summary
CODA is a GPU kernel abstraction introduced by researchers from MIT and other institutions that reframes Transformer block computations as GEMM-plus-epilogue programs. The core insight is that many memory-bound operators (normalization, activations, residual updates) can be algebraically restructured to execute while a GEMM output tile is still on-chip, eliminating costly round-trips to global memory. The system provides a small set of composable epilogue primitives that cover nearly all non-attention computation in standard Transformer forward and backward passes, and works with both human- and LLM-authored kernels.

## Key takeaways
- **Memory bandwidth is a key bottleneck**: Despite heavy optimization of matrix multiplications, surrounding element-wise operations waste significant time moving data through global memory.
- **GEMM-epilogue fusion**: CODA fuses normalization, activations, residual updates, and reductions directly into the GEMM kernel's epilogue, keeping data on-chip and avoiding redundant memory traffic.
- **Composable primitives**: The abstraction exposes a constrained but expressive set of epilogue building blocks (scaling, reductions, pairwise transforms, accumulation) that can be combined without sacrificing GEMM performance.
- **Near-complete coverage**: CODA covers nearly all non-attention computation in both forward and backward passes of a standard Transformer block.
- **LLM-authoring compatible**: Both human-written and LLM-generated CODA kernels achieve high performance, suggesting this abstraction is accessible enough for automated kernel generation workflows.
- **Practical path forward**: The approach bridges framework-level productivity (easy composition) with hardware-level efficiency (expert GEMM performance).
